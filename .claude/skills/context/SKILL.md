# コンテキスト要約

現在のセッションの作業状況を要約し、引き継ぎや振り返りに使えるサマリーを生成します。

## 実行手順

### Step 1: 情報収集

以下の情報を自動で収集：

#### 1-1. Git状態
```bash
git status
git diff --stat
git log --oneline -5
```

#### 1-2. 変更ファイル
- 新規作成されたファイル
- 変更されたファイル
- 削除されたファイル

#### 1-3. 未コミットの変更
- ステージング済み
- 未ステージング

---

### Step 2: コンテキストの整理

以下の形式で要約を生成：

```markdown
## セッションコンテキスト

### 現在のブランチ
`feature/xxx`

### 作業中のタスク
- [タスクの概要]
- 目的: [何を達成しようとしているか]
- 進捗: [どこまで完了したか]

### 変更内容
| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| path/to/file1 | 新規 | [何を追加したか] |
| path/to/file2 | 変更 | [何を変更したか] |

### 未完了タスク
- [ ] [残っている作業1]
- [ ] [残っている作業2]

### 次のステップ
1. [次にやるべきこと]
2. [その次にやるべきこと]

### 注意点・ブロッカー
- [問題があれば記載]
- [確認が必要な点]

### 関連ファイル（参考）
- `path/to/related/file` - [関連性]
```

---

### Step 3: 出力オプション

ユーザーに確認：

「コンテキストを要約しました。どうしますか？
1. 画面に表示するのみ
2. `notes/context/` に保存する
3. クリップボードにコピー（手動）」

---

## 使用場面

- **セッション終了時** - 次回の引き継ぎ用
- **長時間作業後** - 現状把握
- **別の人に引き継ぐ時** - 状況共有
- **コンテキストが混乱した時** - 整理

## 使用例

```
/context
```
