# 技術的負債の検出

セッション終了時や定期的に実行し、コードベースの技術的負債を検出・整理します。

## 実行手順

### Step 1: 対象範囲の確認

ユーザーに確認してください：

「技術的負債の検出を行います。対象範囲を選んでください：
1. 今回のセッションで変更したファイル
2. 特定のディレクトリ（パスを指定）
3. プロジェクト全体（時間がかかります）」

---

### Step 2: 検出項目

以下の観点でコードを分析：

#### 2-1. 重複コード
- 同じ処理が複数箇所に存在
- コピペされたコードブロック
- 共通化できるロジック

#### 2-2. 未使用コード
- 呼び出されていない関数/メソッド
- 使用されていない変数/定数
- 不要なimport文

#### 2-3. 改善候補
- 長すぎる関数（50行以上）
- 深いネスト（3段階以上）
- マジックナンバー
- 不明確な命名

#### 2-4. TODOコメント
- `TODO`, `FIXME`, `HACK`, `XXX` を検索
- 放置されている課題を可視化

---

### Step 3: 結果の報告

以下の形式で報告：

```markdown
## 技術的負債レポート

### 検出サマリー
| 種類 | 件数 | 優先度 |
|------|------|--------|
| 重複コード | X件 | 高/中/低 |
| 未使用コード | X件 | 高/中/低 |
| 改善候補 | X件 | 高/中/低 |
| TODOコメント | X件 | - |

### 詳細

#### 重複コード
1. **[ファイルA] と [ファイルB]**
   - 該当箇所: 行XX-YY
   - 内容: [何が重複しているか]
   - 推奨: [共通化の提案]

#### 未使用コード
1. **[ファイル名]:[行番号]**
   - `function_name()` - 呼び出し元なし
   - 推奨: 削除 or 確認が必要

#### 改善候補
1. **[ファイル名]:[行番号]**
   - 問題: [具体的な問題]
   - 推奨: [改善案]

#### TODOコメント
1. **[ファイル名]:[行番号]**
   - `// TODO: [内容]`
   - 作成日: [わかれば]
```

---

### Step 4: 次のアクション提案

「検出が完了しました。

次のアクションを選んでください：
1. 重複コードを共通化する
2. 未使用コードを削除する
3. 改善候補に対応する
4. 結果を `notes/techdebt/` に保存する
5. 今回は対応しない」

---

## 使用例

```
/techdebt
```

## 自動検出コマンド例

```bash
# TODOコメントの検索
grep -rn "TODO\|FIXME\|HACK\|XXX" --include="*.php" app/

# 未使用のuse文（PHP）
# ※ 静的解析ツール推奨

# 長い関数の検出
# ※ 静的解析ツール推奨
```
